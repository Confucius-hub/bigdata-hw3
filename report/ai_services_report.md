# ДЗ 3, часть 1 — Отчёт по AI-сервисам

Цель — поставить и попробовать четыре AI-инструмента (Cursor, Claude Code, VSCode с AI-плагинами, локальная Gemma через Ollama) и зафиксировать, насколько активно каждый из них использует интернет.

## Тестовая среда

- MacBook на чипе Apple Silicon (M1/M2)
- macOS 14+
- 8 ГБ оперативной памяти

## Установка

| Инструмент | Команда установки |
|---|---|
| Cursor | Скачан DMG с https://cursor.com/download → перетащил в Applications |
| VSCode | `brew install --cask visual-studio-code` |
| Ollama | Скачан DMG с https://ollama.com/download → перетащил в Applications |
| Node.js | `brew install node` |
| Claude Code | `npm install -g @anthropic-ai/claude-code` |

После установки Ollama для локальной Gemma выполнено:

```bash
ollama pull gemma2:9b
ollama run gemma2:9b
```

## Что тестировал

Каждому инструменту задавал одинаковую серию вопросов:

1. **Локальный вопрос по коду:** «Объясни что делает этот файл» — на открытом Python скрипте.
2. **Свежая информация:** «Какая последняя стабильная версия Apache Spark?»
3. **Простая команда:** «Сгенерируй boilerplate для argparse в Python».

## Наблюдения

### Cursor

- Установка: 1 клик, при первом запуске просит войти через GitHub/Google.
- Запросы по локальному коду: моментально использует индекс проекта.
- Свежая информация: модель отвечает «не могу проверить онлайн без явной просьбы», при упоминании `@Web` уходит во внешний поиск.
- **Интернет:** активно используется для каждого AI-действия, потому что сама модель работает в облаке.

### Claude Code

- Установка через npm работает за 30 секунд после Node.js.
- При первом запуске запрашивает Anthropic API-ключ или вход в аккаунт Claude.
- Самый «агентный» из четырёх — может читать, писать, запускать команды без подтверждения каждой мелочи.
- **Интернет:** каждый шаг рассуждения уходит в Anthropic API. Без интернета не работает совсем.

### VSCode + Continue + GitHub Copilot

- VSCode сам по себе без AI просто редактор.
- Установил два расширения:
  - **Continue** — открытое, можно подключить к любому провайдеру (включая локальную Ollama).
  - **GitHub Copilot** — закрытый, требует подписку, работает только через облако GitHub.
- **Интернет:**
  - Copilot — всегда требует.
  - Continue + Ollama — после скачивания модели работает **полностью офлайн**.

### Gemma через Ollama

- Модель `gemma2:9b` скачана один раз (~5 ГБ).
- Запуск: `ollama run gemma2:9b`, дальше обычный чат.
- После загрузки модели **интернет не нужен**. Проверил: отключил Wi-Fi — Gemma продолжает отвечать на вопросы.
- Минус: на M-чипе работает медленнее облачных моделей и хуже справляется со сложными задачами по коду.

## Сравнительная таблица

| Инструмент | Установка | Интернет в работе | Скорость | Автономность от облака |
|---|---|---|---|---|
| Cursor | DMG-установщик + вход | Требуется для каждого AI-действия | Высокая | Нет |
| Claude Code | `npm install` + ключ | Требуется на каждом шаге | Высокая | Нет |
| VSCode + Copilot | расширение + подписка | Требуется всегда | Высокая | Нет |
| VSCode + Continue + Ollama | расширение + Ollama + `ollama pull` | **Только при скачивании модели** | Средняя | **Да** |
| Gemma через Ollama | Ollama + `ollama pull` | **Только при скачивании модели** | Средняя | **Да** |

## Выводы

- Все облачные AI-ассистенты (Cursor, Claude Code, GitHub Copilot) — это, по сути, тонкие клиенты над удалёнными API. Без интернета они неработоспособны.
- Единственный полностью офлайн путь — это `VSCode + Continue + Ollama` либо чистый `ollama run`. Цена — скорость и качество ответов на сложные задачи.
- В повседневной разработке облачные инструменты выигрывают за счёт скорости и качества. Для работы с чувствительными данными или без интернета подходит только локальная Gemma/Llama через Ollama.
